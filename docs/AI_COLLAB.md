# AI 협업 기록

이 문서는 MajorMap AI 개발 과정에서 AI 도구를 어떻게 활용했는지, 그리고 어떤 판단을 사람이 직접 검토하고 확정했는지 기록한다. 공모전 제출 관점에서 중요한 것은 "AI를 사용했다"는 사실 자체보다, 어떤 역할을 AI에 맡기고 어떤 핵심 판단은 사람이 책임졌는지를 분명히 남기는 것이다.

---

## 협업 원칙

MajorMap AI의 AI 협업은 두 층위로 나뉜다.

- 제품 내부 AI 활용: 추천 이유 설명, 부족 역량 해설, 로드맵 문장화에 사용
- 제품 내부 비AI 판단: 추천 점수 계산, 과목 추천, 학기 계획 우선순위 계산은 구조화 데이터 + 규칙 기반으로 처리
- 개발 과정의 AI 활용: 타입 설계, 샘플 데이터 정리, 점수 로직 초안, UI 초안, 문서 초안 작성에 활용
- 사람이 직접 수행한 판단: 최종 범위 결정, 데이터 타당성 검토, 점수 가중치, UX 흐름, 보안 검토

이 원칙은 현재 서비스 구조에도 그대로 반영되어 있다. `OPENAI_API_KEY`가 없거나 LLM 호출이 실패해도 추천과 학기 계획 기능은 계속 동작하며, AI는 설명 보조 계층으로만 사용된다.

## 사람이 직접 검토·판단한 항목

### 1. 데이터 타당성

- 샘플 과목 17개와 진로 4개의 태그 매핑이 실제 커리큘럼 맥락에서 어색하지 않은지 검토
- 복수전공 조합이 특정 진로에 과도한 보너스를 주지 않는지 확인
- 2023/2024 학번 트랙 분리 기준과 전공별 노출 과목 범위 확정

### 2. 점수 로직

- 필수 역량 커버리지와 선택 역량 커버리지의 비중 조정
- 전공 적합도 보너스가 보조 수준으로만 작동하는지 확인
- 관련 과목 수가 적을 때 low-evidence penalty가 실제로 의미 있게 작동하는지 검토
- 핵심 과목 성적 감점과 재수강 권장이 추천 결과를 과하게 뒤집지 않는지 확인

### 3. UX 흐름

- 데모 프로필 A/B/C가 서로 다른 추천 결과를 충분히 보여주는지 확인
- 결과 카드에서 적합도, 근거 충분도, 보유 역량, 부족 역량, 보완 과목 순서가 이해하기 쉬운지 검토
- 추천 결과에서 바로 학기 계획으로 이어지는 흐름이 발표와 시연에 적합한지 검토
- 학기 계획 화면의 남은 학점 안내와 이월 과목 문구가 과도하게 복잡하지 않은지 확인

### 4. 보안 및 안정성

- `OPENAI_API_KEY`가 클라이언트 코드에 노출되지 않는지 확인
- `lib/llm.ts`가 서버 전용으로만 동작하는지 검토
- LLM 호출 실패 시 fallback 설명으로 서비스 흐름이 유지되는지 확인

### 5. 문서와 제출 관점

- README가 공개 저장소 기준으로 과장 없이 현재 구현 범위를 설명하는지 검토
- 공모전 제출 문서에서 실제 구현 범위와 확장 예정 범위를 명확히 구분했는지 확인
- "왜 AI를 이렇게 사용했는가"가 심사자에게 짧게 전달되는 구조인지 점검

## AI 한계와 사람 판단이 필요했던 결정

| 상황 | AI가 잘한 일 | 사람이 최종 판단한 일 |
| --- | --- | --- |
| 과목-진로 태그 매핑 | 초안 구조와 연결 후보를 빠르게 제안 | 실제 교육 맥락에서 타당한지 검토 후 채택 여부 결정 |
| 점수 정규화와 보정 | 커버리지 기반 계산식과 penalty 구조를 제안 | 점수 체감, 결과 분포, 과도한 역전 여부를 검토해 수치 조정 |
| 데모 프로필 구성 | A/B/C 시나리오 초안을 빠르게 생성 | 각 프로필이 발표용으로 충분히 다른 결과를 내는지 직접 검증 |
| LLM 해설 문구 | 설명 초안과 fallback 문장을 빠르게 생성 | 없는 사실을 만들지 않는지, 데모 톤이 과장되지 않는지 검토 |
| README 및 제출 문서 | 문서 구조와 초안을 빠르게 정리 | 심사 관점에서 메시지 우선순위와 실제 구현 범위를 직접 조정 |

AI는 속도와 초안 생성에 강했고, 사람은 맥락 판단과 책임 있는 범위 결정에 집중했다.

## 현재 코드 기준 AI 협업 결과

현재 저장소에서 AI 협업 결과가 반영된 핵심 흐름은 다음과 같다.

- `/recommend`에서 학번, 전공 조합, 수강 과목, 성적, 관심 키워드 입력
- `/api/recommend`에서 규칙 기반 진로 Top 3 계산
- 결과 카드에서 적합도, 추천 확신도, 보유 역량, 부족 역량, 핵심 미이수 과목, 재수강 권장 확인
- `/api/explain`에서 LLM 기반 자연어 해설 제공
- `/api/plan`에서 선택 진로 기준 다음 1~2개 학기 계획 생성
- `OPENAI_API_KEY`가 없어도 fallback 설명으로 전체 흐름 유지

즉, AI는 서비스의 핵심 판단을 대체하지 않고, 구조화된 추천 시스템을 더 이해하기 쉽게 만드는 역할에 머물도록 설계되었다.

## 정리

MajorMap AI에서 AI는 핵심 판단 엔진이 아니라, 구조화된 추천 시스템을 빠르게 구현하고 이해하기 쉽게 전달하기 위한 협업 도구이자 설명 계층으로 사용되었다. 추천 점수와 계획 생성의 기준은 사람이 설계한 구조화 데이터와 규칙 기반 로직에 남겨두고, AI는 초안 생성, 해설 보조, 문서화 생산성을 높이는 방향으로 제한적으로 활용했다.
