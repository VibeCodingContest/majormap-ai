# 프롬프트 로그 양식

AI 협업 흔적을 남기기 위한 로그 양식이다. 작업 단위마다 아래 템플릿을 복사해 `docs/AI_COLLAB.md`에 추가하거나 별도 로그 파일로 관리한다.

---

## 작성 원칙

- AI가 생성한 코드/문서라도 사람이 검토하고 수정한 내용을 반드시 기록한다.
- 프롬프트 전문을 붙여넣을 필요는 없다. 핵심 의도만 요약한다.
- "AI가 틀린 부분", "사람이 수정한 부분"을 구체적으로 적을수록 좋다.

---

## 템플릿

```markdown
## [날짜] [작업 제목]

- **담당**: 성혁 / 준하 / 공동
- **사용 도구**: Cursor Agent (Plan Mode) / Cursor Agent (Agent Mode) / Codex
- **작업 단계**: 1단계(분석) / 2단계(추천 로직) / 3단계(데이터) / 4단계(LLM API) / 5단계(폼 UI) / 6단계(결과 UI) / 7단계(문서)

### 프롬프트 요약
(어떤 목표로 AI에게 요청했는지 2~3줄)

### AI 출력 요약
(AI가 생성한 주요 내용 — 파일 목록, 핵심 로직 등)

### 사람이 검토/수정한 부분
(AI 출력 중 틀렸거나 보완이 필요했던 부분, 사람이 직접 판단한 내용)

### 결과 파일
- `파일경로1` — 변경 요약
- `파일경로2` — 변경 요약

### 남은 이슈
(이 작업 후 추가로 처리해야 할 항목)
```

---

## 작성 예시

```markdown
## 2026-04-07 추천 로직 강화 (2단계)

- **담당**: 성혁
- **사용 도구**: Cursor Agent (Agent Mode)
- **작업 단계**: 2단계(추천 로직)

### 프롬프트 요약
단순 태그 점수 합산에서 벗어나 필수 태그 점수, 선택 태그 점수, 관심 키워드 보너스,
복수전공 보너스를 반영하고 0~100 정규화된 점수를 반환하도록 추천 로직을 강화 요청.
scoreBreakdown과 reasons 필드를 결과에 포함하도록 명시.

### AI 출력 요약
- `lib/scoring.ts` 신규 작성 — calcScore(), buildReasons() 순수 함수
- `lib/recommendation.ts` 리팩터 — scoring.ts 활용, 상위 3개 반환
- `lib/types.ts` — ScoreBreakdown 타입, CareerRecommendation 필드 추가
- `app/api/recommend/route.ts` — 입력 검증 추가

### 사람이 검토/수정한 부분
- MAX_RAW_SCORE 기준값(120)이 진로별 태그 수에 따라 불균형할 수 있음 확인
  → 현재 데모 데이터 범위 내에서는 문제없다고 판단하고 유지
- buildReasons()가 생성하는 문장이 자연스러운지 직접 확인 후 승인

### 결과 파일
- `lib/scoring.ts` — 신규
- `lib/recommendation.ts` — 리팩터
- `lib/types.ts` — ScoreBreakdown, reasons, summary 추가

### 남은 이슈
- 진로 4개 중 솔루션 컨설턴트 점수가 낮게 나오는 경향 있음 → 데이터 보강 필요
```

---

## 로그 파일 관리 방법

1. 작업마다 `docs/AI_COLLAB.md` 하단에 이 양식을 채워서 추가한다.
2. 로그가 길어지면 날짜별로 `docs/logs/YYYY-MM-DD.md` 파일로 분리해도 된다.
3. 공모전 제출 전에 전체 로그를 검토해 불필요한 내용을 정리한다.
