# QA SCENARIOS

제출 전 내부 검증용 시나리오 모음이다. `majormap-ai`의 현재 추천 + explain fallback 흐름이 안정적으로 동작하는지 확인하는 데 목적이 있다.

---

## 데모 시나리오

### A. PM 지향

- 입력: `A — PM 지향 (2024 컴공+경영)`
- 기대 결과:
  - 추천 1위가 `프로덕트 매니저`
  - 비즈니스/전략 계열 강점이 카드에 표시된다.

### B. 백엔드 지향

- 입력: `B — 백엔드 지향 (2023 컴공 중심)`
- 기대 결과:
  - 추천 1위가 `백엔드 개발자`
  - 프로그래밍/시스템 계열 강점이 카드에 표시된다.

### C. 데이터 지향

- 입력: `C — 데이터 지향 (2024 컴공+경영)`
- 기대 결과:
  - 추천 1위가 `데이터 분석가`
  - 데이터/통계 계열 강점이 카드에 표시된다.

---

## 엣지케이스

### 1. explain API 키 없음

- 조건: `OPENAI_API_KEY` 미설정
- 확인:
  - `AI 해설 보기` 클릭 시 전체 UI가 깨지지 않는다.
  - fallback 설명 또는 안전한 메시지가 노출된다.

### 2. 추천 후 explain 실패

- 조건: 네트워크 오류 또는 explain 응답 실패
- 확인:
  - 결과 카드는 그대로 남아 있다.
  - 오류 메시지가 카드 내부에만 제한적으로 표시된다.

### 3. 데모 프로필 전환

- 조건: A 실행 후 B 또는 C로 바로 변경
- 확인:
  - 이전 결과가 새 프로필 기준으로 갱신된다.
  - 혼합된 과목 체크 상태가 남지 않는다.

### 4. 최소 입력 조합

- 조건: 수강 과목과 관심 키워드를 최소 수준으로 입력
- 확인:
  - 추천 API가 500 없이 응답한다.
  - 낮은 점수여도 UI가 비정상 종료되지 않는다.
