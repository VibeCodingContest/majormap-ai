# PROD QA

실제 live URL 기준 최종 수동 점검 문서다. 발표용 문서가 아니라 제출 직전 확인 순서만 정리한다.

---

## 1. 홈 접속

- live URL의 `/` 접속
- 서비스 제목과 설명이 정상 렌더링되는지 확인
- `추천 시작하기` 버튼이 `/recommend`로 이동하는지 확인

## 2. `/recommend` 진입

- 추천 입력 화면이 깨지지 않고 로드되는지 확인
- 데모 프로필 A/B/C 버튼이 보이는지 확인
- 학번, 전공, 관심 키워드 입력 UI가 보이는지 확인

## 3. 추천 결과 확인

- 데모 프로필 A/B/C 각각에서 `진로 추천 받기` 클릭
- Top 3 결과 카드가 렌더링되는지 확인
- 점수, 추천 요약, 보유 역량, 부족 역량, 추천 이유가 보이는지 확인

## 4. explain 성공 시 확인 포인트

- `OPENAI_API_KEY`가 배포 환경에 설정된 경우 `AI 해설 보기` 클릭
- headline, fit summary, 추천 근거, 보완 포인트, 로드맵이 보이는지 확인
- explain 패널을 열고 닫아도 결과 카드 상태가 깨지지 않는지 확인

## 5. explain fallback 시 확인 포인트

- `OPENAI_API_KEY`가 없는 배포 환경에서도 `AI 해설 보기` 클릭
- fallback 설명 또는 안전한 에러 메시지가 카드 내부에서만 표시되는지 확인
- explain 실패가 추천 결과 사용을 막지 않는지 확인

## 6. 모바일 최소 확인 포인트

- 모바일 최소 폭에서 `/recommend` 진입
- 데모 버튼과 추천 버튼이 화면 밖으로 사라지지 않는지 확인
- 결과 카드 텍스트가 겹치지 않는지 확인
- explain 패널이 세로 스크롤 안에서 읽을 수 있는지 확인

## 7. live URL 기준 최종 수동 점검 순서

1. `/` 접속
2. `/recommend` 진입
3. 데모 A 추천 결과 확인
4. 데모 B 추천 결과 확인
5. 데모 C 추천 결과 확인
6. `AI 해설 보기` 확인
7. 모바일 최소 폭 한 번 확인
8. 마지막으로 public repo URL과 live URL이 제출 문서에 반영됐는지 확인
