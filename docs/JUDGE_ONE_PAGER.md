# JUDGE ONE PAGER

## 서비스 한 줄 정의

MajorMap AI는 대학생의 수강 이력과 전공 조합을 바탕으로 적합한 진로를 추천하고, 부족한 역량과 보완 과목을 함께 제시하는 규칙 기반 진로 설계 서비스다.

---

## 기존 진로 추천 서비스와의 차이

기존 진로 추천 서비스는 성향 검사나 흥미 설문 중심인 경우가 많다. MajorMap AI는 학생이 실제로 수강한 과목과 학번별 커리큘럼을 입력값으로 사용한다는 점이 다르다.

또한 결과를 단순 직무명으로 끝내지 않고, 점수 세부 내역, 보유 역량, 부족 역량, 보완 과목을 함께 보여준다. 즉, "무엇을 추천하는가"보다 "왜 그렇게 추천하는가"를 설명하는 구조에 가깝다.

---

## 왜 공모전 주제와 맞는지

이 프로젝트는 AI를 단순 생성 기능이 아니라 교육 맥락의 의사결정 보조 도구로 사용한다.

- 학생의 이수 데이터를 해석해 진로 선택을 돕는다.
- 설명 가능한 추천 구조를 제공한다.
- AI 설명이 실패해도 핵심 추천 기능은 유지된다.
- public repo와 live URL 기준으로 재현 가능한 형태를 갖췄다.

즉, 교육 현장의 실제 문제를 작은 범위에서 끝까지 구현 가능한 구조로 정리했다는 점에서 공모전 취지와 맞는다.

---

## 핵심 차별점 4개

### 1. 수강 이력 기반 추천

학생의 실제 이수 과목에서 태그를 추출해 진로 요구 역량과 비교한다. 단순 흥미 설문보다 현재 학업 이력 반영도가 높다.

### 2. 설명 가능한 결과 구조

추천 결과에 점수 세부 내역, 보유 역량, 부족 역량, 보완 과목, 추천 이유가 함께 제공된다. 결과를 해석할 수 있다는 점이 핵심이다.

### 3. 복수전공과 학번 차이 반영

주전공, 복수전공, 학번 트랙을 입력에 포함해 같은 학과 학생이라도 다른 결과가 나올 수 있도록 설계했다.

### 4. fallback을 전제로 한 안정적 AI 활용

OpenAI API는 설명 보조 기능에만 사용하고, API 키가 없거나 호출이 실패해도 추천 핵심 흐름은 그대로 유지된다.

---

## MVP 범위

현재 실제 구현 범위는 아래와 같다.

- 대학: 1개 대학 가정
- 학과군: 컴퓨터공학, 경영학
- 학번 트랙: 2023학번, 2024학번
- 진로: 백엔드 개발자, 프로덕트 매니저, 데이터 분석가, 솔루션 컨설턴트
- 과목 데이터: 15개 샘플

의도적으로 포함하지 않은 범위는 다음과 같다.

- 실제 학사 시스템 연동
- 시간표 충돌 계산
- 로그인/회원가입
- 전 학과 확장
- 실제 채용 공고 연동

이 MVP는 추천 정확도의 절대 수치보다, 추천 근거를 구조화해 보여주는 흐름 검증에 집중한 버전이다.

## 심사 포인트 대응 문장

### 문제 적합성

"학생이 이미 들은 과목을 진로와 연결해 해석하는 문제"에 집중했다는 점에서 교육 현장의 실제 pain point와 맞닿아 있다.

### 구현 완성도

추천과 explain fallback이 분리된 구조로 구현되어 있으며, `npm run lint`와 `npm run build`를 통과하는 실행 가능한 상태다.

### AI 활용 적절성

AI는 핵심 판단을 대체하지 않고, 설명 생성과 개발 생산성 향상에 제한적으로 사용했다. 핵심 추천 로직은 규칙 기반으로 유지했다.

### 확장 가능성

현재는 작은 데이터 범위로 제한했지만, 과목 데이터와 진로 정의를 확장하면 다른 학과군과 학번 체계로 넓힐 수 있다.

### 제출 안정성

public repo, live URL, AI report PDF, 제출 서류 기준으로 정리 가능하며, `OPENAI_API_KEY`가 없어도 추천 핵심 흐름이 깨지지 않도록 설계했다.
