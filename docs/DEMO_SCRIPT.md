# MajorMap AI 내부 클릭 검증 흐름

이 문서는 발표 스크립트가 아니라, 현재 `majormap-ai` 추천 흐름을 수동으로 확인할 때 따라갈 순서를 정리한 내부 검증 문서다.

---

## 사전 준비

1. `npm run dev` 실행 후 `http://localhost:3000` 접속 확인
2. 브라우저 화면을 `/recommend` 페이지에 맞춰 준비
3. `OPENAI_API_KEY`가 없어도 fallback으로 AI 해설 동작 확인 가능

---

## 시나리오 A — PM 지향 (2024학번, 컴공+경영)

1. `/recommend` 상단에서 **"A — PM 지향 (2024 컴공+경영)"** 버튼 클릭
   - 학번 2024 / 주전공 컴퓨터공학 / 복수전공 경영학 자동 입력
   - 과목 5개 자동 체크: 프로그래밍입문, 경영학원론, 마케팅, 경영전략, IT프로젝트관리
   - 관심 키워드 자동 입력: product, strategy, UX

2. **"진로 추천 받기"** 버튼 클릭

3. 결과 확인
   - **1위: 프로덕트 매니저**
   - 보유 역량 배지: 비즈니스, 전략, 커뮤니케이션
   - 복수전공 보너스 반영 확인

4. 1위 카드 하단 **"AI 해설 보기"** 클릭
   - RoadmapPanel 펼쳐짐
   - headline: "프로덕트 매니저 적합도 ○○점"
   - 추천 근거, 보완 포인트, 학습 로드맵 3단계 확인

---

## 시나리오 B — 백엔드 지향 (2023학번, 컴공 단일)

1. **"B — 백엔드 지향 (2023 컴공 중심)"** 버튼 클릭
   - 학번 2023 / 주전공 컴퓨터공학 / 복수전공 없음 자동 입력
   - 과목 5개 자동 체크: 프로그래밍입문, 데이터구조, 운영체제, 데이터베이스, 소프트웨어공학
   - 관심 키워드: backend, server, API

2. **"진로 추천 받기"** 버튼 클릭

3. 결과 확인
   - **1위: 백엔드 개발자**
   - 보유 역량: 프로그래밍, 알고리즘, 문제해결, 시스템설계
   - 부족 역량: 없음 (필수 역량 충족)
   - 추천 과목: 없음 (부족 역량이 없어 추가 과목 미제안)

4. explain 패널이 안전하게 열리고 닫히는지 확인

---

## 시나리오 C — 데이터 지향 (2024학번, 컴공+경영)

1. **"C — 데이터 지향 (2024 컴공+경영)"** 버튼 클릭
   - 학번 2024 / 주전공 컴퓨터공학 / 복수전공 경영학 자동 입력
   - 과목 5개 자동 체크: 프로그래밍입문, 데이터구조, 데이터분석개론, 통계학, 머신러닝기초
   - 관심 키워드: data, analytics, ML

2. **"진로 추천 받기"** 버튼 클릭

3. 결과 확인
   - **1위: 데이터 분석가**
   - 보유 역량: 데이터분석, 통계, 프로그래밍, 알고리즘
   - 추천 과목: 없음 또는 경영 관련 과목 (business 태그 부재 시)

4. A/B/C 세 가지 결과가 서로 다른 1위 진로를 반환하는지 확인

---

## 확인 포인트

| 포인트 | 화면 위치 |
|---|---|
| 규칙 기반 점수 (LLM 의존 X) | 결과 카드 scoreBreakdown 표 |
| 복수전공 보너스 차이 | A(복수전공 있음) vs B(없음) 점수 비교 |
| AI 해설 fallback 동작 | API 키 없이도 AI 해설 패널 표시 |
| 학번별 과목 필터 | 학번 변경 시 체크박스 목록 변경 |
